**Note on `discrepancy`:** The definition uses `⨆` (`iSup`) over `ℝ`, which is a conditionally complete lattice. If the supremum over centers is unbounded (which it will be for most sequences), `sSup` returns a junk value. This is acceptable for the conjecture statement because the theorem only asserts `discrepancy A r ≥ C * Real.sqrt r`, which is well-defined regardless — if the supremum is truly infinite the inequality holds trivially. The upper bound variant similarly works since the existential quantifier over `A` and `C` makes the bound meaningful when finite.

### `answer()` usage

The original problem asks: "Is $f(r)$ unbounded for every $A$? How fast does $f(r)$ grow?"

The first part is a **yes/no question** (Case 1), while the second asks for quantitative bounds. Since the problem has been **solved** by Beck, the formalization captures the quantitative results directly as assertions (Case 3):
- The main theorem states Beck's lower bound: $f(r) \geq C\sqrt{r}$ for all locally finite $A$.
- The variant states Beck's upper bound: there exists $A$ with $f(r) \leq C\sqrt{r\log r}$.

- [x] Case 3 is appropriate here: the quantitative bounds are direct assertions, and the lower bound implies the yes/no answer (unboundedness). Using `answer()` for the yes/no part alone would be weaker than the actual result.
- [x] No `answer()` wrapper needed since both theorems state direct bounds (not yes/no or find-the-value).
