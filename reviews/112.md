### `answer()` usage

The original problem says "Determine $k(n,m)$", which is a **Case 2** problem — asking for a specific value. The formalization should ideally have a main theorem of the form:

```lean

    dirRamseyNum n m = answer(sorry) := by sorry
```

However, the file instead formalizes the **Erdős–Rado upper bound** as a direct assertion (Case 3 pattern):

```lean

    ∀ n m : ℕ, 2 ≤ n → 2 ≤ m →
      dirRamseyNum n m ≤ (...) := by sorry
```

- [ ] **Issue**: The main problem asks to "determine $k(n,m)$" (a value-finding problem), but the formalization only captures a known upper bound, not the full determination question. The `erdos_112` theorem should either (a) use `= answer(sorry)` to capture the full open problem (Case 2, `research open`), with the Erdős–Rado bound as a variant `erdos_112.variants.erdos_rado_upper_bound` categorized as `research solved`, or (b) at minimum include a variant with the full Case 2 formulation.
