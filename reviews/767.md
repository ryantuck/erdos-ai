### `answer()` usage

The original problem on erdosproblems.com asks: "Is it true that $g_k(n) = (k+1)n - (k+1)^2$ for $n$ sufficiently large?" — a yes/no question ("Is it true that ..."), so **Case 1** applies.

- [ ] **FAIL**: The theorem type should be `answer(True) ↔ P` (since the problem is a yes/no question answered affirmatively by Jiang), but the current formalization uses a direct assertion (Case 3) without `answer()`:
  ```lean
  -- Current (incorrect for a yes/no problem):
  theorem erdos_767 (k : ℕ) (hk : k ≥ 1) :
      ∃ N₀ : ℕ, ∀ n : ℕ, N₀ ≤ n →
        maxEdgesAvoidingCycleChords k n = (k + 1) * n - (k + 1) ^ 2 := by sorry
  ```
  Should be:
  ```lean
  -- Expected (Case 1, solved affirmatively):
  theorem erdos_767 : answer(True) ↔
      ∀ (k : ℕ), k ≥ 1 → ∃ N₀ : ℕ, ∀ n : ℕ, N₀ ≤ n →
        maxEdgesAvoidingCycleChords k n = (k + 1) * n - (k + 1) ^ 2 := by sorry
  ```
- [ ] **FAIL**: Quantifiers `(k : ℕ)` and `(hk : k ≥ 1)` are theorem parameters instead of appearing after `answer()` inside the RHS of `↔`. The linter `linter.style.answer_attribute` enforces this.

## Summary

The file is well-structured with clean definitions, proper docstrings, correct copyright/import/namespace boilerplate, and good mathematical modeling. However, there are two blocking issues related to `answer()` usage:

1. **Missing `answer()` wrapper (§6)**: The original problem asks "Is it true that ...?", which is a yes/no question (Case 1). Since Jiang proved it affirmatively, the theorem should use `answer(True) ↔ P`, not a bare direct assertion.

**Suggested fix:**
```lean
@[category research solved, AMS 5]
theorem erdos_767 : answer(True) ↔
    ∀ (k : ℕ), k ≥ 1 → ∃ N₀ : ℕ, ∀ n : ℕ, N₀ ≤ n →
      maxEdgesAvoidingCycleChords k n = (k + 1) * n - (k + 1) ^ 2 := by sorry
```
