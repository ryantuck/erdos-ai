## Mathematical Review

The theorem formalizes:

> ¬ ∃ (k : ℕ) (a : Fin k → ℤ) (n : Fin k → ℕ),
>   (∀ i, 2 ≤ n i) ∧ (Function.Injective n) ∧
>   (∀ x : ℤ, ∃! i, (↑(n i) : ℤ) ∣ (x - a i))

This states: there is no finite collection of congruence classes with distinct moduli ≥ 2 such that every integer belongs to exactly one class. This matches the classical Mirsky-Newman / Davenport-Rado theorem and correctly captures the problem from erdosproblems.com/947.

**Edge cases are handled correctly:**
- `k = 0`: `∃!` over `Fin 0` is vacuously false, so excluded.
- `k = 1`: a single modulus ≥ 2 cannot cover every integer exactly once, so excluded.

The formalization is mathematically faithful.
