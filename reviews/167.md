### `answer()` usage

- [ ] **VIOLATION — Wrong case for `answer()`.** The original problem on erdosproblems.com is phrased as a yes/no question: *"If G is a graph with at most k edge disjoint triangles then **can** G be made triangle-free after removing at most 2k edges?"* The word "can" makes this a Case 1 problem. Per the checklist:

  > If it asks a yes/no question — "asked whether", "is it true that", "does there exist", **"can one"** — use Case 1.

  The theorem should use the `answer(sorry) ↔ P` form:
  ```lean
  theorem erdos_167 : answer(sorry) ↔
      ∀ {V : Type*} [Fintype V] [DecidableEq V]
        (G : SimpleGraph V) [DecidableRel G.Adj] (k : ℕ),
      (∀ (T : Finset (Finset V)),
        (∀ t ∈ T, G.IsNClique 3 t) →
        (∀ t₁ ∈ T, ∀ t₂ ∈ T, t₁ ≠ t₂ → (t₁ ∩ t₂).card ≤ 1) →
        T.card ≤ k) →
      ∃ (H : SimpleGraph V),
        H ≤ G ∧
        H.CliqueFree 3 ∧
        (G.edgeSet \ H.edgeSet).ncard ≤ 2 * k := by sorry
  ```

- [ ] **VIOLATION — Quantifiers as theorem parameters instead of after `answer()`.** The checklist requires: *"Quantifiers go **after** `answer(sorry)`, not as theorem parameters."* Currently `{V : Type*}`, `(G : SimpleGraph V)`, `(k : ℕ)`, and `(hpack : ...)` are all theorem parameters. They should be universally quantified on the RHS of the `↔`.
