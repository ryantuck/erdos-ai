## Mathematical Correctness Notes

The formalization is mathematically sound:

- **`Embeds`**: Defines graph embedding as an injective adjacency-preserving map. This is the standard subgraph (not induced subgraph) notion, which is correct for Ramsey numbers. As a `Prop`, it does not need `noncomputable`.

- **`ramseyNum`**: Uses `sInf` on the set of $N$ such that every 2-coloring of $K_N$ (modeled as any `SimpleGraph (Fin N)` and its complement) contains either a copy of $G$ or a copy of $H$. This correctly defines the two-color Ramsey number $R(G,H)$. Edge case: `sInf ∅ = 0` for `ℕ`, but the Ramsey number always exists for finite graphs, so the set is nonempty in practice.

- **`cycleGraph`**: Adjacency `i ≠ j ∧ (j.val = (i.val + 1) % m ∨ i.val = (j.val + 1) % m)` correctly defines the cycle $C_m$ on $m$ vertices. The `symm` and `loopless` proofs are correct term-mode proofs.

- **"No isolated vertices"**: Formalized as `∀ v : Fin n, ∃ w, H.Adj v w` — every vertex has at least one neighbor. Correct.

- **"Sufficiently large $m$"**: Captured by `∃ M₀ : ℕ, ... H.edgeSet.ncard ≥ M₀ →`, where $M_0$ depends on $k$ but not on the specific graph $H$. Correct.

- **Floor division**: `(k - 1) / 2` is natural number division, which is floor division. Matches $\lfloor(k-1)/2\rfloor$. Correct.

- **`H.edgeSet.ncard`**: Uses `ncard` (cardinality for potentially infinite sets, returning `ℕ`). For `SimpleGraph (Fin n)`, the edge set is finite, so `ncard` correctly counts edges.
