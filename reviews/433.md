### `answer()` usage

- [ ] **FAIL — Wrong case for `answer()`.** The module docstring states: *"Is it true that g(k,n) ~ n²/(k-1)?"* — this is a yes/no question ("Is it true that..."), which falls under **Case 1**. The problem was proved by Dixmier, so the answer is affirmative. The theorem should use `answer(True) ↔ P`, not a bare assertion (Case 3). Current:
  ```lean
  theorem erdos_433 (k : ℕ) (hk : 2 ≤ k) :
      (fun n : ℕ => (g433 k n : ℝ)) ~[atTop]
      (fun n : ℕ => (n : ℝ) ^ 2 / ((k : ℝ) - 1)) := by
    sorry
  ```
  Expected (Case 1, solved affirmatively):
  ```lean
  theorem erdos_433 : answer(True) ↔
      ∀ k : ℕ, 2 ≤ k →
        (fun n : ℕ => (g433 k n : ℝ)) ~[atTop]
        (fun n : ℕ => (n : ℝ) ^ 2 / ((k : ℝ) - 1)) := by
    sorry
  ```

- [ ] **FAIL — Quantifiers as theorem parameters.** `(k : ℕ)` and `(hk : 2 ≤ k)` are theorem parameters. With `answer()`, quantifiers must go **after** `answer(...)`, not as theorem parameters. The `∀ k, 2 ≤ k →` should be on the RHS of the `↔`.
