### `answer()` usage

The original problem on erdosproblems.com asks: "Does there always exist a Steiner system $S(r,k,n)$, provided the trivial necessary divisibility conditions are satisfied...?" and the module docstring mirrors this phrasing. This is a **yes/no question** (Case 1) — it begins with "Does there always exist..." Per the checklist:

> If it asks a yes/no question — "asked whether", "is it true that", "does there exist", "can one" — use Case 1.

Since the problem has been solved affirmatively by Keevash [Ke14], the correct form per the checklist is:

```lean
theorem erdos_722 : answer(True) ↔
    ∀ (r k : ℕ), r ≥ 1 → k > r →
      ∃ N₀ : ℕ, ∀ n : ℕ, n ≥ N₀ →
        SteinerDivisibilityConditions r k n →
        ∃ F : Finset (Finset (Fin n)), IsSteinerSystem r k n F := by sorry
```

- [ ] **Non-conformance:** The theorem currently uses a direct assertion (Case 3 style) with `r` and `k` as theorem parameters, rather than `answer(True) ↔ P` (Case 1). The original problem is phrased as a yes/no question, so Case 1 applies.
- [ ] **Non-conformance:** Quantifiers `(r k : ℕ)`, `(hr : r ≥ 1)`, and `(hkr : k > r)` are theorem parameters instead of being placed after the biconditional. The checklist rule "Quantifiers go **after** `answer(sorry)`, not as theorem parameters" would apply once Case 1 is adopted.
