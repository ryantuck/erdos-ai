## Mathematical Accuracy Notes

The formalization uses 0-indexed stars (`Fin s`, `Fin t`) whereas the source uses 1-indexed ($i \leq s$, $j \leq t$). This is a valid re-indexing:

- **Source:** $k$ ranges over $\{2, \ldots, s+t\}$ (since $i \geq 1, j \geq 1$), giving $s+t-1$ values.
- **Formalization:** $k$ ranges over $\{0, \ldots, s+t-2\}$ via `Finset.range (s + t - 1)`, also $s+t-1$ values.
- The degree sequences and pairs are the same up to relabeling, so the formula is preserved.

The `twoColorSizeRamseyNumber` definition correctly captures the size Ramsey number $\hat{R}(G_1, G_2)$: it takes the infimum over edge counts of host graphs $H$ such that every symmetric 2-coloring of vertex pairs yields a monochromatic copy of $G_1$ (in color `true`) or $G_2$ (in color `false`). The coloring is over all vertex pairs (not just edges), but since the embedding conditions require adjacency in $H$, non-edge colors are irrelevant — the definition is equivalent to the standard one.

The `lFun` definition uses `Finset.sup` over `ℕ` (with `⊥ = 0`), correctly returning the maximum of $n_i + m_j - 1$ over valid pairs. Natural number subtraction truncation is not an issue since the hypotheses `hn_pos` and `hm_pos` guarantee $n_i + m_j \geq 2$.
