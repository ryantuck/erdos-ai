### `answer()` usage

The original problem asks "Is there a sequence of positive integers n_k, m_k such that n_k/m_k → α and σ(n_k) = σ(m_k)?" — a yes/no question ("Is there ..."), so **Case 1** applies. The problem has been solved affirmatively by Pollack [Po15b].

- [ ] **FAIL**: Theorem type should be `answer(True) ↔ P` (Case 1, solved affirmatively), but is currently a direct assertion `∃ (n m : ℕ → ℕ), ...` (Case 3 style). The original problem is phrased as a yes/no question ("Is there a sequence..."), not a direct assertion. Per the checklist: "Read the problem's English description... If it asks a yes/no question — 'asked whether', 'is it true that', 'does there exist', 'can one' — use Case 1."
- [ ] **FAIL**: Quantifiers `(α : ℝ) (hα : α ≥ 1)` are theorem parameters, but with `answer()` form they must go **after** `answer(True)`, not as theorem parameters. The correct form would be:
  ```lean
  theorem erdos_823 : answer(True) ↔
      ∀ (α : ℝ), α ≥ 1 →
        ∃ (n m : ℕ → ℕ), (∀ k, 0 < n k) ∧ (∀ k, 0 < m k) ∧
        (∀ k, Nat.sumDivisors (n k) = Nat.sumDivisors (m k)) ∧
        Tendsto (fun k => (n k : ℝ) / (m k : ℝ)) atTop (nhds α) := by sorry
  ```
- [x] The RHS (`P`) correctly formalizes the mathematical content: for all α ≥ 1, existence of sequences n_k, m_k with positivity, equal σ-values, and convergence of the ratio to α.
- [x] `answer(True)` would mean "yes" (such sequences exist) to the original question.
