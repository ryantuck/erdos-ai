### `answer()` usage

The module docstring says: *"Is it true that |A| ≤ (1/2 + o(1))·N?"* — a yes/no question ("Is it true that..."), so **Case 1** applies.

- [ ] **Issue.** The main theorem `erdos_635` is stated as a direct assertion (Case 3 style):
  ```lean
  theorem erdos_635 (t : ℕ) (ht : t ≥ 1) :
      ∀ ε : ℝ, ε > 0 → ... := by sorry
  ```
  Per the checklist, since the problem asks "Is it true that...", this should use `answer(True) ↔ P` (Case 1, solved affirmatively):
  ```lean
  theorem erdos_635 : answer(True) ↔
      ∀ t : ℕ, t ≥ 1 → ∀ ε : ℝ, ε > 0 →
        ∃ N₀ : ℕ, ∀ N : ℕ, N ≥ N₀ →
          ∀ A : Finset ℕ, A ⊆ Finset.Icc 1 N →
            DivAvoidance A t → (A.card : ℝ) ≤ (1 / 2 + ε) * (N : ℝ) := by sorry
  ```
  The docstring notes the problem has been "resolved affirmatively," which is consistent with `answer(True)`.
- [ ] **Issue (related).** `t` and `ht` are theorem **parameters**, but the checklist requires quantifiers to go **after** `answer(sorry)` (inside the statement), not as parameters. Currently:
  ```lean
  theorem erdos_635 (t : ℕ) (ht : t ≥ 1) : ∀ ε ...
  ```
  Should be:
  ```lean
  theorem erdos_635 : answer(True) ↔ ∀ t : ℕ, t ≥ 1 → ∀ ε ...
  ```
- [x] Variant theorems (`t_eq_one`, `t_eq_two_lower`) are direct assertions of known results for specific values of t — Case 3 is appropriate since they state concrete equalities/bounds, not yes/no questions.
