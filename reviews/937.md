### `answer()` usage

The original problem is phrased as: *"Are there infinitely many four-term arithmetic progressions of coprime powerful numbers?"* — this is a yes/no question ("Are there..."), so **Case 1** applies.

The problem has been proved in the affirmative by Bajpai–Bennett–Chan [BBC24], so the theorem should use `answer(True) ↔ P`.

- [ ] **FAIL.** The theorem is currently stated as a direct assertion (Case 3 style):
  ```lean
  theorem erdos_937 :
      ∀ N : ℕ, ∃ a d : ℕ, N ≤ a ∧ 0 < a ∧ 0 < d ∧ ... := by sorry
  ```
  It should instead use the `answer(True) ↔ P` pattern (Case 1, solved affirmatively):
  ```lean
  theorem erdos_937 :
      answer(True) ↔
        ∀ N : ℕ, ∃ a d : ℕ, N ≤ a ∧ 0 < a ∧ 0 < d ∧
          IsPowerful a ∧ IsPowerful (a + d) ∧
          IsPowerful (a + 2 * d) ∧ IsPowerful (a + 3 * d) ∧
          Nat.Coprime a (a + d) ∧
          Nat.Coprime a (a + 2 * d) ∧
          Nat.Coprime a (a + 3 * d) ∧
          Nat.Coprime (a + d) (a + 2 * d) ∧
          Nat.Coprime (a + d) (a + 3 * d) ∧
          Nat.Coprime (a + 2 * d) (a + 3 * d) := by sorry
  ```
- [x] Quantifiers are inside the theorem body, not as theorem parameters. (Correct even after the fix above.)
