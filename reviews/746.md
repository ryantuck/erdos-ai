### `answer()` usage

- [ ] **ISSUE — The original problem is phrased as a yes/no question, but no `answer()` wrapper is used.**

  The source at [erdosproblems.com/746](https://www.erdosproblems.com/746) asks:

  > *"Is it true that, almost surely, a random graph on n vertices with ≥ (1/2+ε)n log n edges is Hamiltonian?"*

  Per the checklist §6 ("How to decide which case applies"), "is it true that" triggers **Case 1**. Since the problem has been proved affirmatively, the theorem should use:

  ```lean
  theorem erdos_746 : answer(True) ↔
      ∀ ε : ℝ, ε > 0 →
      ∀ δ : ℝ, δ > 0 →
      ∃ n₀ : ℕ, ∀ n : ℕ, n ≥ n₀ →
        ∀ m : ℕ, (m : ℝ) ≥ (1/2 + ε) * (n : ℝ) * Real.log (n : ℝ) →
          m ≤ n.choose 2 →
          gnmFraction n m (fun G => G.IsHamiltonian) ≥ 1 - δ := by
    sorry
  ```

  The current formulation uses Case 3 (direct assertion without `answer()`), which is appropriate when the problem says "Show that..." or "Prove that..." but not when it asks "Is it true that...?"

- [x] Quantifiers are inside the theorem type (not as parameters). No `answer()` parameters to check, but if `answer(True)` is added, quantifiers would correctly remain on the RHS.
