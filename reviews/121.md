### `answer()` usage

- [ ] **The original problem is a yes/no question but the formalization uses Case 3 instead of Case 1.**

The module docstring states: *"Conjectured by Erdős, Sós, and Sárközy: Is F₅(N) = (1 - o(1))N? More generally, is F_{2k+1}(N) = (1 - o(1))N for all k ≥ 2?"* This is phrased as a yes/no question ("Is ...?"), which per the checklist should use **Case 1**: `answer(False) ↔ P`.

The current formalization instead states Tao's quantitative disproof result directly (Case 3):
```lean
theorem erdos_121 :
    ∀ k : ℕ, 4 ≤ k →
    ∃ c : ℝ, 0 < c ∧ ...
```

Strictly per the checklist, since the problem *asks* a yes/no question that was answered negatively, it should be:
```lean
theorem erdos_121 : answer(False) ↔
    (∀ k : ℕ, 2 ≤ k →
      ∀ᶠ N : ℕ in atTop, (F (2 * k + 1) N : ℝ) / N → 1) := by sorry
```
where the RHS formalizes the original question ("Is F_{2k+1}(N) = (1-o(1))N for all k ≥ 2?") so that `answer(False)` means "no."

**Note:** The current formalization is mathematically informative — it captures Tao's stronger result for all k ≥ 4 (not just odd k ≥ 5). This is a reasonable mathematical choice, but it does not conform to the checklist's `answer()` conventions for yes/no problems. Furthermore, the formalized statement is about a *different* (stronger) claim than the original conjecture: the original asks about odd k = 2k+1 for k ≥ 2, while the formalization covers all k ≥ 4.

**Recommendation:** Either (a) reformulate using `answer(False) ↔ P` where P is the original conjecture, possibly adding the current Tao result as a variant, or (b) add a justification comment explaining why the direct-assertion form was chosen over the `answer()` form.
