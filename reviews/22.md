The original problem asks: "Is there a graph on n vertices with ≥ n²/8 many edges which contains no K₄ such that the largest independent set has size at most εn?" This is a yes/no question ("Is there..."), so **Case 1** applies.

- [ ] **FAIL:** The theorem type should be `answer(True) ↔ P` (since the problem is solved affirmatively), but is instead stated as `P` directly (Case 3 style). Per the checklist: "If it asks a yes/no question — 'asked whether', 'is it true that', 'does there exist', 'can one' — use Case 1." The correct form would be:
  ```lean

      ∀ ε : ℝ, ε > 0 →
        ∃ N : ℕ, ∀ n : ℕ, n ≥ N →
          ∃ G : SimpleGraph (Fin n),
            (n : ℝ) ^ 2 / 8 ≤ (G.edgeSet.ncard : ℝ) ∧
            (∀ s : Finset (Fin n), ¬G.IsNClique 4 s) ∧
            (∀ s : Finset (Fin n), G.IsIndepSet ↑s → (s.card : ℝ) ≤ ε * (n : ℝ)) := by
    sorry
  ```
- N/A: Quantifier placement rule — quantifiers are already in the theorem body, not as parameters.

**Mitigating context:** The module docstring also gives an equivalent reformulation as a direct assertion ("rt(n; 4, εn) ≥ n²/8 for sufficiently large n"), and the theorem docstring frames the result as a proved fact. One could argue the formalization follows the equivalent assertive form (Case 3).
