### `answer()` usage

The module docstring states: "Does every graph on n vertices with more than ex(n; C₄) edges contain ≫ n^{1/2} many copies of C₄?" This is a **yes/no question** ("Does ..."), so per the checklist this is **Case 1** and the theorem should use `answer(sorry) ↔ P`.

- **[FAIL]** The theorem type is a direct assertion (Case 3 style):
  ```lean
  theorem erdos_60 :
      ∃ (c : ℝ) (_ : c > 0) (N₀ : ℕ), ∀ n : ℕ, N₀ ≤ n → ... := by sorry
  ```
  But the problem is phrased as "Does every graph ... contain ≫ n^{1/2} copies of C₄?", which is a yes/no question. Per checklist §6 Case 1, the theorem should be:
  ```lean
  theorem erdos_60 : answer(sorry) ↔
      ∃ (c : ℝ) (_ : c > 0) (N₀ : ℕ),
      ∀ n : ℕ, N₀ ≤ n →
      ∀ G : SimpleGraph (Fin n),
        G.edgeSet.ncard > extremalNumber60 (cycleGraph60 4 (by omega)) n →
        (labeledC4Count n G : ℝ) ≥ c * (n : ℝ) ^ ((1 : ℝ) / 2) := by sorry
  ```
- [x] Quantifiers are not passed as theorem parameters (they are all in the body). Correct regardless of which case applies.
