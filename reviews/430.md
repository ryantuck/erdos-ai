## Mathematical Fidelity

The formalization is faithful to the original problem:

1. **Sequence definition:** `seq n 0 = n - 1` correctly encodes $a_1 = n - 1$ (index shift from 1-based to 0-based). `nextTerm n prev` selects the greatest $m \in [2, \text{prev})$ with all prime factors $> n - m$, matching the original.

2. **Search range `[2, prev)` vs `[1, prev)`:** The original uses $[1, a_{k-1})$. Excluding $m = 1$ is harmless: `Nat.minFac 1 = 1`, so `n < 1 + 1` is false for any $n \geq 2$, meaning $m = 1$ would never satisfy the filter for relevant $n$.

3. **Prime factor condition:** `n < m + m.minFac` is equivalent to `m.minFac > n - m`. Since `minFac` returns the smallest prime factor for $m \geq 2$, this correctly captures "all prime factors of $m$ exceed $n - m$."

4. **Termination:** When no valid $m$ exists, `nextTerm` returns `0`. Subsequent calls stay at `0`. The theorem's condition `2 ≤ seq n k` correctly restricts attention to active (non-terminated) terms.

5. **Conjecture statement:** `∃ N, ∀ n ≥ N, ∃ k, 2 ≤ seq n k ∧ ¬(seq n k).Prime` correctly formalizes "for sufficiently large $n$, not all terms of the sequence are prime" — i.e., there exists an active term that is composite.
