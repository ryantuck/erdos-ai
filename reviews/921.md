### `answer()` usage

- [ ] **The original problem is a yes/no question** ("Is it true that $f_k(n) \asymp n^{1/(k-2)}$?"). Per the checklist (Case 1), the main theorem should use `answer(True) ↔ P` format since the problem has been solved affirmatively. Instead, the formalization decomposes the asymptotic equivalence into two direct assertions (lower bound + upper bound), each stated as Case 3 (direct assertion). This is mathematically reasonable and arguably clearer, but does not conform to the checklist's guidance for yes/no questions.
- [ ] **Quantifiers as theorem parameters**: `(k : ℕ) (hk : k ≥ 4)` are theorem parameters. Per the checklist, when `answer()` is used, quantifiers should go **after** `answer()`, not as parameters. Since `answer()` is not used, this rule is not technically violated, but it would become an issue if the formalization were restructured to use `answer()`.

**Suggested fix** (to strictly follow the checklist):
```lean
@[category research solved, AMS 5]
theorem erdos_921 : answer(True) ↔
    ∀ k : ℕ, k ≥ 4 →
      (∃ C : ℝ, C > 0 ∧ ∃ N₀ : ℕ, ∀ n : ℕ, n ≥ N₀ →
        (erdos921F k n : ℝ) ≥ C * (n : ℝ) ^ ((1 : ℝ) / ((k : ℝ) - 2))) ∧
      (∃ C : ℝ, C > 0 ∧ ∃ N₀ : ℕ, ∀ n : ℕ, n ≥ N₀ →
        (erdos921F k n : ℝ) ≤ C * (n : ℝ) ^ ((1 : ℝ) / ((k : ℝ) - 2))) := by
  sorry
```

Alternatively, the current decomposition into lower/upper bounds is acceptable if the project convention permits stating solved yes/no questions as direct assertions. The lower bound alone as `erdos_921` is slightly misleading since it only captures half of the original problem.
