### `answer()` usage

- [ ] **FAIL — Wrong case / missing `answer()`.** The original problem on erdosproblems.com states: *"Estimate f(n,k) for k > n²/4."* The docstring itself says: *"Erdős asks whether this bound can be improved when the number of edges exceeds n²/4."*

  This is either a Case 2 problem ("Estimate …" → use `= answer(sorry)`) or a Case 1 problem ("asks whether" → use `answer(sorry) ↔ P`). Either way, `answer()` is needed. The current theorem instead formalizes only the **known baseline result** (Erdős–Goodman–Pósa: f(n,k) ≤ ⌊n²/4⌋ for all k) as a direct assertion (Case 3), which does not capture what Problem 1017 actually asks.

  **What the formalization should capture:** The open question is whether f(n,k) < ⌊n²/4⌋ when k > n²/4. For example, as a Case 1 yes/no formulation:
  ```lean
  theorem erdos_1017 : answer(sorry) ↔
      ∀ᶠ n in Filter.atTop,
        ∀ (G : SimpleGraph (Fin n)) (dG : DecidableRel G.Adj),
          haveI := dG;
          G.edgeFinset.card > n ^ 2 / 4 →
            ∃ (k : ℕ) (...), k < n ^ 2 / 4 ∧ ... := by sorry
  ```
  Or as a Case 2 "estimate" formulation providing a bound as a function of n and k.

## Summary

**Two blocking issues:**

1. **Wrong problem formalized (§6 `answer()` usage).** The theorem states the Erdős–Goodman–Pósa result (f(n,k) ≤ ⌊n²/4⌋ for all graphs), which is a known theorem from 1966. Erdős Problem 1017 asks whether this bound can be *sharpened* when k > n²/4. The formalization needs to capture the actual open question using `answer()` (Case 1 or Case 2), not just the baseline result.

2. **Wrong category (§7).** Tagged `research solved` but the problem is listed as **open** on erdosproblems.com. Should be `research open`. (The current formalization of the Erdős–Goodman–Pósa result *is* solved, but that's not Problem 1017.)

**Suggestion:** Keep the current theorem as a helper lemma (e.g., `erdos_1017.variants.erdos_goodman_posa` with `category research solved`) and add a new main theorem `erdos_1017` that formalizes the actual open question with `answer(sorry)` and `category research open`.
