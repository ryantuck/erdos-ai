### `answer()` usage

The original problem on erdosproblems.com asks: "Does there exist a constant $c > 0$ such that $\max_{|z|=1} |P(z)| \geq (1+c)\sqrt{n}$?" — this is a yes/no question ("Does there exist..."), so **Case 1** applies.

The problem has been **disproved** (answered negatively by Kahane). Per the checklist, the formalization should be:

```lean
theorem erdos_230 : answer(False) ↔ P := by sorry
```

where `P` formalizes the original question as asked (i.e., `∃ c : ℝ, 0 < c ∧ ∀ n ≥ 2, ∀ polynomial with unimodular coefficients, max|P(z)| ≥ (1+c)√n`), and `answer(False)` indicates the answer is "no".

- [ ] **FAIL**: The theorem does not use `answer()` at all. Instead, it directly asserts the negation of the conjecture (Kahane's result) as a Case 3 direct assertion. The current statement is:
  ```lean
  theorem erdos_230 :
      ∀ ε : ℝ, 0 < ε →
      ∃ N₀ : ℕ, ∀ n : ℕ, N₀ ≤ n →
      ∃ a : Fin n → ℂ,
        (∀ k : Fin n, ‖a k‖ = 1) ∧
        ∀ z : ℂ, ‖z‖ = 1 →
          ‖∑ k : Fin n, a k * z ^ (k.val + 1)‖ ≤ (1 + ε) * Real.sqrt ↑n := by
    sorry
  ```
  This formalizes the *negation* of the original question directly. Per the checklist, yes/no questions must use the `answer(sorry) ↔ P` pattern (or `answer(True)`/`answer(False)` when solved), with `P` formalizing the question **as originally asked**.
- [ ] **FAIL**: Quantifiers should go after `answer(...)` (inside the RHS of `↔`), not as the top-level theorem type. The `answer_attribute` linter enforces this.

**Suggested fix:**
```lean
theorem erdos_230 : answer(False) ↔
    ∃ c : ℝ, 0 < c ∧ ∀ n : ℕ, 2 ≤ n →
    ∀ a : Fin n → ℂ, (∀ k : Fin n, ‖a k‖ = 1) →
    ∃ z : ℂ, ‖z‖ = 1 ∧
      (1 + c) * Real.sqrt ↑n ≤ ‖∑ k : Fin n, a k * z ^ (k.val + 1)‖ := by
  sorry
```
