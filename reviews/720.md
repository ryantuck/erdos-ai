### `answer()` usage

The original problem asks three yes/no questions ("Is it true that $\hat{R}(P_n)/n \to \infty$?", "Is it true that $\hat{R}(P_n)/n^2 \to 0$?", "Is it true that $\hat{R}(C_n) = o(n^2)$?"). Under a strict reading these would be **Case 1** (yes/no questions requiring `answer(True/False) ↔ P`).

However, the formalization instead captures Beck's stronger result — that the size Ramsey numbers are linear — as a **direct assertion** (Case 3). This is a reasonable modeling choice: the linear bound immediately resolves all three original questions, and the docstrings clearly explain how. Formalizing the stronger result is arguably more useful than formalizing the weaker original questions with `answer()` wrappers.

- [x] Theorem type is a direct assertion (`∃ C : ℕ, ∀ n, ...`) — no `answer()` wrapper. Appropriate for Case 3 formulation.
- [x] No misplaced quantifiers as theorem parameters — all quantifiers are in the theorem body.

**Note:** A strict interpretation of the checklist could prefer three separate theorems with `answer(False) ↔ ...` (question 1) and `answer(True) ↔ ...` (questions 2 and 3). The current approach of formalizing Beck's stronger result as a direct assertion is a defensible alternative that captures strictly more information.
