### `answer()` usage

- [ ] **FAIL:** The original problem is phrased as a yes/no question ("Must $A$ contain the vertices of ...?"), so **Case 1** applies. The theorems should use the `answer(True) ↔ P` form with quantifiers on the RHS, not the current direct-assertion form (Case 3).

  Current (all four theorems):
  ```lean
  theorem erdos_353
      (A : Set (ℝ × ℝ))
      (hA_meas : MeasurableSet A)
      (hA_inf : volume A = ⊤) :
      ∃ p₁ p₂ p₃ p₄ : ℝ × ℝ, ... := by sorry
  ```

  Expected (Case 1, solved affirmatively):
  ```lean
  theorem erdos_353 : answer(True) ↔
      ∀ (A : Set (ℝ × ℝ)), MeasurableSet A → volume A = ⊤ →
      ∃ p₁ p₂ p₃ p₄ : ℝ × ℝ, ... := by sorry
  ```

  This applies to all four theorems (`erdos_353` and all three variants), since each docstring phrases the problem as "Must $A$ contain...?"

- [ ] **FAIL:** Quantifiers (`A`, `hA_meas`, `hA_inf`) are theorem parameters instead of being placed after the `answer()` / `↔`. Per the checklist: "Quantifiers go **after** `answer(sorry)`, not as theorem parameters."

## Summary

**2 issues found, both related to `answer()` usage:**

1. **Missing plain-English description in module docstring** (§3): The docstring has a heading, reference, and bibliography but lacks a brief summary of the problem.

2. **Incorrect `answer()` form** (§6): All four theorems formalize a yes/no question ("Must A contain...?") as a direct assertion (Case 3) instead of using `answer(True) ↔ P` (Case 1). Since the problems are solved affirmatively, the correct form is `answer(True) ↔ (∀ A, MeasurableSet A → volume A = ⊤ → ∃ ...)`. The universal quantifiers over `A`, `hA_meas`, and `hA_inf` should be on the RHS of `↔`, not as theorem parameters.

**Everything else passes:** copyright header, import, namespace, custom definitions (docstrings, `noncomputable` markings, naming conventions), category/AMS attributes, proof bodies, variant naming, and code quality are all conformant.
