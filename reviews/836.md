### `answer()` usage

**Issue flagged.** The original problem poses two yes/no questions:

1. *"Must G contain O(r²) many vertices?"* — answered negatively by Alon.
2. *"Must there be two edges which meet in ≫ r many vertices?"* — remains open.

The file formalizes only the second question, which is reasonable since the first has been disproved. However, the second question is phrased as "Must there be…" — a yes/no question that falls under **Case 1**. Per the checklist, the statement should use:

```lean
theorem erdos_836 : answer(sorry) ↔
    ∃ C : ℝ, C > 0 ∧ ∀ r : ℕ, r ≥ 2 → ... := by sorry
```

Instead, the file states the result as a **direct assertion (Case 3)** without `answer()`:

```lean
theorem erdos_836 :
    ∃ C : ℝ, C > 0 ∧ ∀ r : ℕ, r ≥ 2 → ... := by sorry
```

This asserts that the answer is "yes" (there must be two edges meeting in Ω(r) vertices), but since the problem is open, the answer is unknown and should be `answer(sorry)`.

**Positive note**: All quantifiers are inside the theorem statement (not as theorem parameters), which is correct and would satisfy `linter.style.answer_attribute` if `answer()` were added.
