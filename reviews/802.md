### `answer()` usage

- [ ] **FAIL — Wrong case applied.** The original problem on erdosproblems.com is phrased as a yes/no question:

  > *"Is it true that any $K_r$-free graph on $n$ vertices with average degree $t$ contains an independent set on $\gg_r \frac{\log t}{t}n$ many vertices?"*

  This is **Case 1** ("Is it true that..."), so the theorem should use `answer(sorry) ↔ P`, not a bare assertion. Currently the theorem is:

  ```lean
  theorem erdos_802 (r : ℕ) (hr : r ≥ 3) : ∃ c : ℝ, c > 0 ∧ ... := by sorry
  ```

  It should instead be:

  ```lean
  theorem erdos_802 : answer(sorry) ↔
      ∀ (r : ℕ), r ≥ 3 →
      ∃ c : ℝ, c > 0 ∧ ... := by sorry
  ```

- [ ] **FAIL — Quantifiers as theorem parameters.** Per the checklist: *"Quantifiers go after `answer(sorry)`, not as theorem parameters."* Currently `r : ℕ` and `hr : r ≥ 3` are theorem parameters. Under the corrected Case 1 form, they must be universally quantified inside the RHS of the `↔`.

## Summary

**Status: Changes Requested**

The formalization is mathematically clean and well-structured. The custom `avgDegree` definition is properly documented and marked `noncomputable`, the independent set condition is correctly expressed, and the `CliqueFree r` condition accurately captures the $K_r$-free hypothesis.

**Two issues must be fixed:**

1. **`answer()` form (Case 1):** The original problem is a yes/no question ("Is it true that..."), so the theorem must use `answer(sorry) ↔ P`, not a bare assertion.
2. **Quantifier placement:** `r` and `hr` must move from theorem parameters into the universally quantified body after `answer(sorry) ↔`.

**Minor note:**

- The formalization adds the condition `avgDegree G ≥ 2`, which is not explicitly stated in the original problem (the original just says "average degree $t$"). This is mathematically reasonable — the bound $\frac{\log t}{t}$ is only meaningful for $t > 1$ — but worth flagging for author confirmation.
