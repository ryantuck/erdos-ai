### `answer()` usage

The module docstring states: "The remaining open question is **whether** Λ(k,3) is finite for all odd k ≥ 5." The word "whether" indicates a yes/no question, so **Case 1** applies.

- [ ] **FAIL**: The theorem should use `answer(sorry) ↔ P` form (Case 1), but instead uses a direct assertion (Case 3). Current:
  ```lean
  theorem erdos_436 (k : ℕ) (hk : 5 ≤ k) (hodd : k % 2 = 1) :
      LambdaFinite k 3 := by
    sorry
  ```
  Expected:
  ```lean
  theorem erdos_436 : answer(sorry) ↔
      ∀ k : ℕ, 5 ≤ k → k % 2 = 1 → LambdaFinite k 3 := by
    sorry
  ```
- [ ] **FAIL**: Quantifiers `(k : ℕ)`, `(hk : 5 ≤ k)`, `(hodd : k % 2 = 1)` are theorem parameters instead of being inside the proposition after `answer(sorry)`. The checklist and `linter.style.answer_attribute` require quantifiers to go after `answer(sorry)`.

The variant `erdos_436.variants.hildebrand` states a known solved result (Hildebrand proved it), so it correctly uses Case 3 (direct assertion) — no `answer()` needed. However:

- [ ] **FAIL**: The variant also has quantifiers as theorem parameters `(k : ℕ) (hk : 2 ≤ k)`. Since this is a solved result stated as a direct assertion (Case 3), theorem parameters are acceptable and no `answer()` is needed. However, for consistency with the style of universally quantified statements in this repo, the quantifiers could be internalized. **Non-blocking note** — Case 3 does not explicitly prohibit theorem parameters.
