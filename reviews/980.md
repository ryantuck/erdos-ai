### `answer()` usage

- [ ] **FAIL.** The original problem is phrased as a yes/no question: *"Is it true that ∑ n_k(p) ∼ c_k · x / log x?"* Per the checklist, this is **Case 1**. The formalization should use `answer(True) ↔ P` (since the problem is solved affirmatively by Erdős and Elliott), but instead it directly asserts `P` (Case 3 style).

  Current:
  ```lean
  theorem erdos_980 (k : ℕ) (hk : k ≥ 2) :
      ∃ c : ℝ, c > 0 ∧ ... := by sorry
  ```

  Expected:
  ```lean
  theorem erdos_980 : answer(True) ↔
      ∀ k : ℕ, k ≥ 2 →
      ∃ c : ℝ, c > 0 ∧ ... := by sorry
  ```

- [ ] **FAIL (related).** `k` and `hk` are theorem parameters. If `answer()` is used, quantifiers must go **after** `answer(...)` in the RHS, not as theorem parameters. The linter `linter.style.answer_attribute` would enforce this once `answer()` is added.
