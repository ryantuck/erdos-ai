### `answer()` usage

- [ ] **ISSUE — Missing `answer()` wrapper.** The original problem is phrased as a yes/no question: *"Are there ≫ max_i (a_{i+1} − a_i) many even integers of the form a_{j+1} − a_j?"* Per the checklist, "Are there..." maps to **Case 1**. The theorem should use `answer(sorry) ↔ P`, not a bare assertion `P`. Current:
  ```lean
  theorem erdos_854 :
      ∃ C : ℝ, C > 0 ∧ ∃ K₀ : ℕ, ∀ k : ℕ, k ≥ K₀ →
        ((gapValues (primorial k)).card : ℝ) ≥ C * (maxGap (primorial k) : ℝ) := by
    sorry
  ```
  Should be:
  ```lean
  theorem erdos_854 : answer(sorry) ↔
      ∃ C : ℝ, C > 0 ∧ ∃ K₀ : ℕ, ∀ k : ℕ, k ≥ K₀ →
        ((gapValues (primorial k)).filter (Even ·)).card ≥ C * (maxGap (primorial k) : ℝ) := by
    sorry
  ```
  Note: Erdős himself expressed doubt about this question ("perhaps this is false"), reinforcing that it is a genuine open question rather than a direct assertion.

- [x] Quantifiers are in the theorem body, not as parameters — correct placement.

### Mathematical Fidelity

- [ ] **ISSUE — Even gaps not filtered.** The problem specifically asks about *even integers* among the consecutive gaps. The definition `gapValues` returns *all* distinct gap values (via `(consecutiveDiffs (coprimeList n)).toFinset`) without filtering for evenness. The theorem's docstring correctly says "distinct **even** integers" but the Lean statement counts **all** gap values. The `gapValues` definition or the theorem statement should filter to only even values, e.g.:
  ```lean
  ((gapValues (primorial k)).filter (Even ·)).card
  ```
