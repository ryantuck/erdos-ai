### `answer()` usage

The original problem on erdosproblems.com/206 says Erdős and Graham "asked whether" for almost all $x > 0$ the best underapproximations are eventually greedy — this is a **yes/no question** ("asked whether ..."), so **Case 1** should apply. The problem has been **disproved** (answered negatively), so the expected formalization would be:

```lean
theorem erdos_206 : answer(False) ↔
    (∀ᵐ x ∂volume.restrict (Set.Ioi 0), IsEventuallyGreedy x) := by sorry
```

However, the actual formalization uses **Case 3** (direct assertion):

```lean
theorem erdos_206 :
    volume {x : ℝ | 0 < x ∧ IsEventuallyGreedy x} = 0 := by sorry
```

- [ ] **Does not use `answer()` at all.** The formalization states Kovač's result directly rather than wrapping the original yes/no question with `answer(False) ↔ P`. The docstring explicitly notes: "We formalize Kovač's result (the negation of the original conjecture)."

  **Justification / mitigating factors:** Kovač's result is *strictly stronger* than a "no" answer to the original question. The original conjecture was "almost all $x$ are eventually greedy" (i.e., the complement has measure zero). Saying `answer(False)` to that would only assert "it is not the case that almost all $x$ are eventually greedy." Kovač proved the much stronger statement that the set of eventually greedy $x$ itself has Lebesgue measure zero. The direct assertion captures this stronger result faithfully, which a Case 1 `answer(False) ↔ P` could not. This is a deliberate and well-documented choice, but it does deviate from the checklist's `answer()` format.
