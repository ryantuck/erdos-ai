### `answer()` usage

- [ ] **NON-CONFORMANCE.** The original problem is phrased as a yes/no question: *"Is the sum of reciprocals of distinct cycle lengths … minimised by the complete bipartite graph?"* The theorem docstring also uses *"Is it true that …?"* phrasing. Per the checklist (Case 1), this should use `answer(sorry) ↔ P`:

  ```lean
  theorem erdos_65 : answer(sorry) ↔
      ∀ (n : ℕ) (G : SimpleGraph (Fin n)) (a b : ℕ), a + b = n → [DecidableRel G.Adj] →
        a * b = G.edgeFinset.card → ... := by sorry
  ```

  Currently the theorem is formalized as a direct assertion (Case 3) without `answer()`. While the author's reformulation into an assertion in the docstring is mathematically sound, the checklist rule is: *"Read the problem's English description … If it asks a yes/no question — 'is it true that' — use Case 1."*

- [ ] **Related:** If `answer()` were used, quantifiers (`n`, `G`, `a`, `b`, etc.) must go **after** `answer(sorry)`, not as theorem parameters. Currently they are theorem parameters, which would violate the `linter.style.answer_attribute` rule under an `answer()` formulation.
