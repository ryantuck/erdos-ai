### `answer()` usage

The original problem (module docstring and erdosproblems.com/411) asks: "For which n and r is it true that g_{k+r}(n) = 2·g_k(n) for all large k?" — a "For which" / "determine" question. Per the checklist decision rule, this is **Case 2** ("What is", "Find", "Determine").

- [ ] **The formalization uses Case 3 (direct assertion) instead of Case 2 (`= answer(sorry)`).** The theorem takes `n` and `r` as parameters and directly asserts a conclusion (Cambie's conjecture), rather than using `answer()` to represent the sought characterization.

  **Justification / Mitigation:** The theorem formalizes *Cambie's specific conjecture* — a proposed answer to the open question — as a direct assertion. This is a defensible choice: Cambie's conjecture is a concrete mathematical claim ("if the doubling condition holds, then r = 2 and n = 2^l · p for p ∈ {2,3,5,7,35,47}"), and Case 3 is appropriate for such direct claims. However, strictly following the checklist's decision rule ("For which" → Case 2), the formalization should characterize the solution set via `answer()`, e.g.:

  ```lean
  theorem erdos_411 :
      {p : ℕ × ℕ | 0 < p.1 ∧ 0 < p.2 ∧
        ∃ K, ∀ k, K ≤ k → g^[k + p.2] p.1 = 2 * g^[k] p.1} = answer(sorry) := by sorry
  ```

- [ ] **One-directional formalization.** The current theorem only captures the *necessity* direction: if the doubling condition holds, then (n, r) has a specific form. It does not assert *sufficiency*: that those specific (n, r) pairs actually satisfy the doubling condition. A complete characterization (as implied by "for which") would need both directions. At minimum, a variant asserting the converse (or known verified cases like n = 10, n = 94 for r = 2) would strengthen the formalization.
