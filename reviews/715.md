### `answer()` usage

- [ ] **The source problem is phrased as yes/no questions** — "Does every regular graph of degree 4 contain a regular subgraph of degree 3?" and "Is there any r such that every regular graph of degree r must contain a regular subgraph of degree 3?" Per the checklist (Case 1), "Does..." questions should be formalized as:
  ```lean
  theorem erdos_715 : answer(True) ↔
      ∀ (V : Type*) [Fintype V] [DecidableEq V]
        (G : SimpleGraph V) [DecidableRel G.Adj],
        G.IsRegularOfDegree 4 →
        ∃ H : G.Subgraph,
          H.verts.Nonempty ∧
          ∀ v : H.verts, H.degree v = 3 := by
    sorry
  ```
  Since the problem has been solved affirmatively by Tashkinov [Ta82], the answer should be `answer(True)`. Currently the theorem uses Case 3 (direct assertion) with universally quantified parameters. The checklist explicitly routes "Does..." questions to Case 1.
- [ ] **Quantifiers as theorem parameters**: The current formalization places `{V : Type*} [Fintype V] [DecidableEq V]`, `(G : SimpleGraph V)`, `[DecidableRel G.Adj]`, and `(hreg : ...)` as theorem parameters rather than inside the statement body. If `answer()` is added, the checklist requires all quantifiers to go **after** `answer(...)` (linter `linter.style.answer_attribute` enforces this). Even without `answer()`, placing the universe quantification inside the statement body is more consistent with the intended Case 1 pattern.

**Same issue applies to the variant** `erdos_715.variants.r_geq_5`: it derives from the second question ("Is there any r..."), so it should also use `answer(True) ↔ P` with quantifiers in the body.

### Required fixes

1. **`answer()` wrapper (§6):** The source problem asks "Does every regular graph of degree 4 contain a regular subgraph of degree 3?" — a yes/no question. Per the checklist (Case 1), both the main theorem and variant should use `answer(True) ↔ P` (since both are solved affirmatively), with all quantifiers moved inside the statement body (after `answer(True) ↔`). The `linter.style.answer_attribute` linter will reject universally quantified theorem parameters when `answer()` is present.

### Recommendations (non-blocking)

2. **Docstring/code consistency**: The `erdos_715` docstring states the formalization uses `H.coe` and `IsRegularOfDegree`, but the actual code uses `∀ v : H.verts, H.degree v = 3`. Consider aligning the two — either use `H.coe.IsRegularOfDegree 3` in the code, or update the docstring to reflect the actual formalization.
3. **Second question formalization**: The original problem's second question ("Is there any r...") could be formalized as a separate variant (e.g., `erdos_715.variants.exists_r`) with `∃ r, ∀ ...`. The current variant for r ≥ 5 is mathematically stronger and a good inclusion, but doesn't directly answer the existential question as posed.
