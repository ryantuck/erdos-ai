### `answer()` usage

The original Erdős problem 278 has **two parts** (per erdosproblems.com):

1. *"What is the maximum density of integers covered by a suitable choice of congruences?"* — This is a **Case 2** ("What is...") question, still **open**. It is not formalized in this file.
2. *"Is the minimum density achieved when all the $a_i$ are equal?"* — This is a **Case 1** ("Is...?") yes/no question, **settled** affirmatively by Simpson [Si86].

**Concern:** The formalization addresses part 2 (Simpson's theorem) as a direct assertion (Case 3 style), but the original problem phrases it as a yes/no question ("Is the minimum density achieved when...?"). Strictly following the checklist:

- [ ] Since part 2 is phrased as a yes/no question and has been answered affirmatively, the theorem should use `answer(True) ↔ P` (Case 1 with solved answer). The current formulation states the result directly as `(∀ offsets, ...) ∧ (∀ a, ...)` without an `answer()` wrapper.

**Concern:** The open part 1 (maximum covering density) is mentioned in the docstring as "remains unknown" but is **not formalized** as a separate theorem. Ideally:

- [ ] There should be an additional theorem (or variant) formalizing the open maximum-density question, e.g.:
  ```lean
  @[category research open, AMS 11]
  theorem erdos_278.variants.max_density (moduli : Finset ℕ) (h : ∀ n ∈ moduli, 0 < n) :
      Finset.sup' ... = answer(sorry) := by sorry
  ```

- [x] Quantifiers (`moduli`, `h`) are theorem parameters, which is correct for Case 3 (no `answer()`). If refactored to use `answer()`, quantifiers should move after the `↔`.
