### `answer()` usage

The original problem on erdosproblems.com says *"Let $n_k$ be minimal such that…"* and asks to **determine $n_k$**. This is nominally a **Case 2** ("find a value") problem. However, the formalization explicitly scopes to the **existence question** posed in [Er75h]: *"Does $n_k$ exist?"* — and existence has been proved [MaRo15].

- [x] The theorem is stated as a direct assertion (Case 3): `∃ n : ℕ, ∀ P, ...` — no `answer()` wrapper. This is appropriate because the formalization asserts a fact ("for every $k ≥ 3$, $n_k$ exists") rather than asking for a specific value.
- [x] No `answer()` is used, consistent with Case 3.
- [x] `k` appears as a theorem parameter `(k : ℕ) (hk : k ≥ 3)` rather than a quantifier. The "quantifiers go after `answer(sorry)`" rule only applies when `answer()` is present, so this is acceptable.

> **Note:** The full problem on erdosproblems.com is marked **OPEN** because the exact determination of $n_k$ remains unsolved — only the existence with bound $n_k \ll k^9$ is known. The formalization captures the solved existence sub-problem and correctly uses `category research solved`. A variant formalizing the quantitative bound (e.g., `∃ C, ∀ k, n_k ≤ C * k ^ 9`) or the full "determine $n_k$" question with `= answer(sorry)` could be added in the future.
