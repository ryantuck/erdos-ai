### `answer()` usage

The original problem asks: *"Is it true that, in any finite colouring of the integers, there must be two integers $x \ne y$ of the same colour such that $x + y$ is a square?"* This is a **yes/no question** ("Is it true that..."), so **Case 1** applies.

The problem has been solved affirmatively (Khalfalah–Szemerédi), so the correct form is:

```lean
theorem erdos_439 : answer(True) ↔
    ∀ (c : ℕ) (f : ℕ → Fin c), ∃ x y : ℕ, x ≠ y ∧ f x = f y ∧ IsSquare (x + y) := by
  sorry
```

- [ ] **FAIL.** The theorem is currently stated as a direct assertion (Case 3 style) with no `answer()` wrapper. Since the original problem is phrased as "Is it true that..." it should use `answer(True) ↔ P`.
- [ ] **FAIL.** Quantifiers `(c : ℕ) (f : ℕ → Fin c)` are theorem parameters instead of appearing after `answer()`. Per the checklist: *"Quantifiers go after `answer(sorry)`, not as theorem parameters."*

The same issues apply to the variant `erdos_439.variants.kth_powers` — the follow-up *"What about a $k$-th power?"* is also a yes/no question. The correct form would be:

```lean
theorem erdos_439.variants.kth_powers : answer(True) ↔
    ∀ (k : ℕ), 2 ≤ k → ∀ (c : ℕ) (f : ℕ → Fin c),
      ∃ x y : ℕ, x ≠ y ∧ f x = f y ∧ ∃ z : ℕ, z ^ k = x + y := by
  sorry
```

- [ ] **FAIL.** Variant also omits `answer()` and uses parameters instead of quantifiers.

## Summary

**Status: NEEDS REVISION**

### Blocking issues

1. **Missing `answer()` wrapper (§6, Case 1).** The original problem is phrased as a yes/no question ("Is it true that..."), and it has been solved affirmatively. Both `erdos_439` and `erdos_439.variants.kth_powers` should use `answer(True) ↔ P` rather than stating `P` as a direct assertion.

2. **Quantifiers as theorem parameters (§6, common rules).** Both theorems bind the universally quantified variables (`c`, `f`, and for the variant `k`, `hk`) as theorem parameters. When `answer()` is used, all quantifiers must appear on the RHS of the `↔`, not as theorem-level parameters.

### Note

- The original problem mentions "integers" (`ℤ`), but the formalization uses natural numbers (`ℕ`). The theorem docstring correctly says "natural numbers" rather than "integers," so this is internally consistent. Since the result for `ℕ` implies the result for `ℤ` (any colouring of `ℤ` restricts to a colouring of `ℕ`), this is a reasonable modeling choice, but worth flagging for awareness.
