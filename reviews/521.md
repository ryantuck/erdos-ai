### `answer()` usage

- [ ] **FAIL:** The original problem on erdosproblems.com asks a **yes/no question**: *"Is it true that, almost surely, lim R_n / log n = 2/π?"* This is **Case 1**, requiring the form `answer(sorry) ↔ P`. The current formalization uses Case 3 (direct assertion without `answer()`), which is incorrect for a yes/no question.

  Current (wrong):
  ```lean
  theorem erdos_521 {Ω : Type*} ... (hRad : ...) (hIndep : ...) :
      ∀ᵐ ω ∂μ, Tendsto ... := by sorry
  ```

  Should be:
  ```lean
  theorem erdos_521 : answer(sorry) ↔
      ∀ {Ω : Type*} [MeasurableSpace Ω] {μ : Measure Ω} [IsProbabilityMeasure μ]
        {ε : ℕ → Ω → ℝ},
      (∀ k, IsRademacher μ (ε k)) →
      iIndepFun ε μ →
      ∀ᵐ ω ∂μ, Tendsto
        (fun n => (numRealRoots (signPoly (fun k => ε k ω) n) : ℝ) / Real.log (n : ℝ))
        atTop (nhds (2 / Real.pi)) := by sorry
  ```

- [ ] **FAIL:** Quantifiers go **after** `answer(sorry)`, not as theorem parameters. Currently the probability space, Rademacher, and independence hypotheses are theorem parameters, but with `answer()` they must be quantified on the RHS of `↔`.
