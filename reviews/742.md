### `answer()` usage

- [ ] **FAIL — wrong case pattern.** The original problem on erdosproblems.com is phrased as a yes/no question: *"Is it true that G has at most n²/4 edges?"* This is **Case 1** ("Is it true that …"). Since the problem has been solved affirmatively (proved by Füredi), the theorem should use `answer(True) ↔ P`, not a direct assertion. Current code uses the Case 3 (direct assertion) pattern instead.

  **Current:**
  ```lean
  theorem erdos_742 (n : ℕ) (G : SimpleGraph (Fin n)) ... :
      G.edgeFinset.card ≤ n ^ 2 / 4 := by sorry
  ```

  **Expected (Case 1, solved affirmatively):**
  ```lean
  theorem erdos_742 : answer(True) ↔
      ∀ (n : ℕ) (G : SimpleGraph (Fin n)),
        G.Connected → G.diam = 2 →
        (∀ v w : Fin n, G.Adj v w →
          ¬(G.deleteEdges {Sym2.mk (v, w)}).Connected ∨
          2 < (G.deleteEdges {Sym2.mk (v, w)}).diam) →
        G.edgeFinset.card ≤ n ^ 2 / 4 := by sorry
  ```

- [ ] **FAIL — quantifiers as theorem parameters.** `n`, `G`, `hconn`, `hdiam`, and `hcrit` are all theorem parameters. Under Case 1, all quantifiers must go **after** `answer(...)`, not as theorem parameters. The `linter.style.answer_attribute` would enforce this if `answer()` were present.
