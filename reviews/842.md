### `answer()` usage

- [ ] **FAIL — The original problem is phrased as a yes/no question ("Does G have chromatic number at most 3?") but the formalization uses a direct assertion (Case 3) instead of `answer(True) ↔ P` (Case 1).**

  The checklist states: *"How to decide which case applies: Read the problem's English description. If it asks a yes/no question — 'asked whether', 'is it true that', 'does there exist', 'can one' — use Case 1."*

  The source at erdosproblems.com/842 uses "Does G have chromatic number at most 3?" — this is a yes/no question answered affirmatively. Per the checklist, the theorem should be:

  ```lean
  theorem erdos_842 : answer(True) ↔
      ∀ (n : ℕ), n ≥ 1 →
      ∀ (σ : Equiv.Perm (Fin (3 * n))),
      IsSingleCycle σ →
      (∀ u v : Fin (3 * n),
        (cycleGraphOfPerm σ).Adj u v → ¬(triangleGraph n).Adj u v) →
      (triangleGraph n ⊔ cycleGraphOfPerm σ).chromaticNumber ≤ 3 := by sorry
  ```

- [ ] **FAIL — Quantifiers are theorem parameters instead of inside the statement.** The checklist rule says: *"Quantifiers go after `answer(sorry)`, not as theorem parameters."* Currently `n`, `hn`, `σ`, `hcycle`, and `hnew` are all theorem parameters. If `answer()` is used, these must be universally quantified in the RHS of the `↔`.

## Mathematical Correctness Notes

The formalization accurately captures the problem:

- `triangleGraph n` correctly models n vertex-disjoint triangles on `Fin (3 * n)`, with vertices `3i`, `3i+1`, `3i+2` forming triangle `i`.
- `IsSingleCycle σ` correctly defines a Hamiltonian cycle via a permutation that is a single cycle (every vertex reachable from every other by iteration).
- `cycleGraphOfPerm σ` correctly defines the edge relation of the cycle: `u` and `v` are adjacent iff `σ(u) = v` or `σ(v) = u`.
- The condition `hnew` correctly enforces that the Hamiltonian cycle edges are disjoint from the triangle edges ("all new edges").
- The conclusion `(triangleGraph n ⊔ cycleGraphOfPerm σ).chromaticNumber ≤ 3` correctly states 3-colorability of the union graph.
- The hypothesis `hn : n ≥ 1` is not strictly necessary (the statement is vacuously true for `n = 0`), but is a reasonable modeling choice matching the problem's implicit assumption.
