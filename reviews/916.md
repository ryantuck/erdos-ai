### `answer()` usage

The original problem asks: "Does every graph with $n$ vertices and $2n-2$ edges contain a cycle and another vertex adjacent to three vertices on the cycle?" — this is a **yes/no question** (Case 1: "Does ..."). The module docstring further confirms: "The answer is yes, as proved by Thomassen [Th74]."

- [ ] **FAIL.** Theorem type should be `answer(True) ↔ P` (Case 1, solved affirmatively), but the theorem is instead stated as a direct assertion `P := by sorry` (Case 3 style). The "Does..." phrasing of the original problem requires `answer()` usage.
- [ ] **FAIL.** Quantifiers `(n : ℕ)`, `(hn : n ≥ 4)`, `(G : SimpleGraph (Fin n))`, `[DecidableRel G.Adj]`, and `(hedge : ...)` are theorem parameters. Per the common rules for `answer()` usage, all quantifiers must go **after** `answer(True)` on the RHS of `↔`, not as theorem parameters. The linter `linter.style.answer_attribute` enforces this.
- [ ] **FAIL.** Since the problem is solved (answer is yes), the theorem should use `answer(True)` (not `answer(sorry)`).

**Expected form:**

```lean
@[category research solved, AMS 5]
theorem erdos_916 : answer(True) ↔
    ∀ (n : ℕ), n ≥ 4 →
    ∀ (G : SimpleGraph (Fin n)) (_ : DecidableRel G.Adj),
    G.edgeFinset.card ≥ 2 * n - 2 →
    ∃ (k : ℕ) (cycle : Fin (k + 3) → Fin n),
      Function.Injective cycle ∧
      (∀ i : Fin (k + 3), G.Adj (cycle i)
        (cycle ⟨(i.val + 1) % (k + 3), Nat.mod_lt _ (by omega)⟩)) ∧
      ∃ v : Fin n, v ∉ Set.range cycle ∧
        ∃ (i j l : Fin (k + 3)), i ≠ j ∧ j ≠ l ∧ i ≠ l ∧
          G.Adj v (cycle i) ∧ G.Adj v (cycle j) ∧ G.Adj v (cycle l) := by
  sorry
```
