### `answer()` usage

- [ ] **Case determination and `answer()` conformance.**
  > **Issue:** The docstring says *"The conjecture asked whether for almost all α, the sequence f(αn) is uniformly distributed in [0,1)."* The phrase "asked whether" indicates a **yes/no question** (**Case 1**). Since Schmidt disproved the conjecture, the correct formalization per the checklist should be:
  > ```lean
  > theorem erdos_492 : answer(False) ↔
  >     ∀ (a : ℕ → ℕ), StrictMono a → (∀ i, 0 < a i) →
  >       Tendsto (fun i => (a (i + 1) : ℝ) / (a i : ℝ)) atTop (nhds 1) →
  >       ∀ᵐ α ∂(volume : Measure ℝ),
  >         IsEquidistributed (fun n => fractionalPosition a (α * (n : ℝ))) := by sorry
  > ```
  > where the RHS formalizes the original question as asked (so `answer(False)` means "no"). The current formalization instead directly asserts the negation (∃ counterexample), which is **Case 3** style. While mathematically equivalent, the checklist's decision procedure says: *"If it asks a yes/no question — 'asked whether' ... — use Case 1."* The `answer()` wrapper is missing entirely.
