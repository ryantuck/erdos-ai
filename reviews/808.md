### `answer()` usage

- [ ] **Case determination is debatable.**
  > **Issue:** The source problem on erdosproblems.com is phrased as a direct assertion/conjecture, not as a yes/no question. Per the checklist, a direct assertion that has been disproved should use **Case 3**: `theorem erdos_808 : ¬ P := by sorry`. The file instead uses **Case 1** (`answer(False) ↔ P`). If the original Erdős phrasing was "asked whether…" then Case 1 is correct; if it was a direct conjecture then Case 3 (`¬ P`) is more appropriate. Verify against the primary source.
- [x] If Case 1 is intended: `answer(False) ↔ P` correctly encodes "the answer is no" for a disproved conjecture.
- [x] The RHS formalizes the question as originally stated (so `answer(True)` would mean "yes").
- [x] Quantifiers are **after** `answer(sorry)` on the RHS of `↔` — not as theorem parameters.
