### `answer()` usage

The source at erdosproblems.com phrases this as a yes/no question: *"Is τ(G) ≤ n − H(n) for every graph G on n vertices?"* — and Erdős himself remarked the conjecture might be "completely wrongheaded." This suggests **Case 1** (`answer(sorry) ↔ P`) may be more appropriate.

However, the theorem docstring (line 66) phrases it as a direct assertion: *"If G is a graph on n vertices then τ(G) ≤ n − H(n)"*. The checklist allows looking at the module docstring, theorem docstring, or source reference to decide the case.

- [ ] **Potential Case 1 / Case 3 mismatch.** The formalization uses Case 3 (direct assertion, no `answer()` wrapper). The source reference phrases the problem as a question ("Is τ(G) ≤ n − H(n)?"), and the original author expressed genuine doubt about the truth of the conjecture. Per checklist §6: *"If it asks a yes/no question — 'asked whether', 'is it true that', 'does there exist', 'can one' — use Case 1."* This would require:
  ```lean
  theorem erdos_151 : answer(sorry) ↔
      ∀ (n : ℕ) (G : SimpleGraph (Fin n)),
        cliqueTransversalNumber G ≤ n - H n := by sorry
  ```
  The current direct-assertion form is defensible if the docstring's phrasing is taken as authoritative, but the source website's question phrasing favors Case 1.

## Mathematical Correctness Notes

The formalization correctly captures the problem:

- **Clique transversal number** (`cliqueTransversalNumber`): Defined as `sInf` of cardinalities of sets that intersect every maximal clique with ≥2 vertices. Matches the standard definition of τ(G).
- **Independence number** (`independenceNumber`): Defined as `sSup` of cardinalities of independent sets. Standard definition of α(G).
- **H(n)**: Defined as `sInf { independenceNumber G | G triangle-free on Fin n }`, i.e., the minimum independence number over triangle-free graphs on n vertices. This is equivalent to "the maximum k such that every triangle-free graph on n vertices has an independent set of size k," as stated in the docstring.
- **Main inequality**: `cliqueTransversalNumber G ≤ n - H n` for all n and G on n vertices matches the source problem.
- **ℕ subtraction**: `n - H n` uses natural number subtraction, which is safe since H(n) ≤ n always holds (the independence number of any graph on n vertices is at most n).
