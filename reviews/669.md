### `answer()` usage

- [x] No `answer()` is used. The formalization is **Case 3** (direct assertion): each theorem asserts a fact (existence of a limit, an upper bound).

**Note:** The original problem says "determine $\lim F_k(n)/n^2$ and $\lim f_k(n)/n^2$", which could be read as Case 2 ("determine a value"), suggesting the theorems should use `= answer(sorry)` to capture the unknown limit values. However, the formalization instead asserts that the limits *exist* (an existential statement), which is itself a meaningful open question and fits Case 3. This is a defensible design choice — the existence of the limit is the primary open question, and determining the exact value would be a natural follow-up variant. No change strictly required, but authors could consider adding a Case 2 variant that captures the limit value via `answer(sorry)`.

- [x] Quantifiers: `k` and `hk` appear as theorem parameters in `erdos_669` and `erdos_669.variants.littleF_limit_exists`. Since no `answer()` is used, the "quantifiers after answer" rule does not apply. The `trivial_bound` variant also takes `k`, `hk`, `n` as parameters — acceptable for a Case 3 direct assertion.
