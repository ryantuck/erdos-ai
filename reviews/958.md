**Note on `IsEquidistantCollinear`:** The definition requires `∀ a ∈ A, ∃ i : Fin A.card, a = p + (i.val : ℝ) • v`, which means every point of `A` is *some* element of the arithmetic progression `{p, p+v, …, p+(n−1)v}`, but does not require every index `i` to be realized. So `A` could be a proper subset of the full progression. The standard meaning of "equidistant points on a line" is that `A` is exactly the full set of `n` equally-spaced points. Consider adding a surjectivity condition (a bijection between `A` and `Fin A.card` via the indexing) to tighten this. As written, the definition is *weaker* than intended, which makes `¬IsEquidistantCollinear A` *harder* to prove and makes the theorem *stronger* than intended — so this is at least a safe direction for a counterexample existence statement.
