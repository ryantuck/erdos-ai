## 5. Formalizability

The problem as stated on erdosproblems.com — "estimate the growth of $\sum f(\{\alpha n_k\})$" and "is it $o(N\sqrt{\log\log N})$?" — is **precise enough to formalize**, and the little-o formulation via ε is standard and correct.

**Ambiguity level: Low.** The only ambiguity is whether f is assumed to have zero mean (see Correctness below), which is implicit in the standard analytic number theory convention but not stated on the website.

## 6. Correctness

### 6a. The little-o encoding is correct
The statement
$$\forall \varepsilon > 0,\; \forall^{\mathrm{a.e.}}\, \alpha,\; \exists N_0,\; \forall N \geq N_0,\; |\textstyle\sum_{k<N} f(\{\alpha n_k\})| \leq \varepsilon \cdot N \cdot \sqrt{\log\log N}$$
correctly encodes $o(N\sqrt{\log\log N})$.

Note: the quantifier order has ε outside the a.e. quantifier, so the null set may depend on ε. This is equivalent to the natural reading (ε inside) by taking a countable intersection over ε = 1/n.

### 6b. CRITICAL: Missing zero-mean hypothesis
**The formalization is almost certainly missing the condition $\int_0^1 f(x)\,dx = 0$.**

If $f$ has nonzero mean $\mu = \int_0^1 f \neq 0$, then by standard equidistribution results for lacunary sequences, for a.e. $\alpha$:
$$\sum_{k < N} f(\{\alpha n_k\}) \sim \mu N$$
which grows linearly — far exceeding $N\sqrt{\log\log N}$. Thus the conjecture as formalized (without zero-mean) is **trivially false** for any $f$ with $\int f \neq 0$.

The standard formulation in the literature (e.g., Erdős 1949, Kac 1946, Gaposhkin) studies the fluctuations of $\sum f(\{\alpha n_k\})$ for functions with $\int_0^1 f = 0$. Erdős's lower bound construction in [Er49d] also works with zero-mean functions.

**Recommended fix:** Add the hypothesis
```lean
∫ x in (0 : ℝ)..1, f x = 0
```
or equivalently
```lean
∫ x in Set.Icc (0 : ℝ) 1, f x ∂volume = 0
```

### 6c. Minor: sum indexing
The formalization sums over `k ∈ Finset.range N` (i.e., k = 0, …, N−1), while the website uses $\sum_{1 \leq k \leq N}$ (i.e., k = 1, …, N). This is an inconsequential index shift for the asymptotic statement.

### 6d. Behavior at small N
`Real.sqrt (Real.log (Real.log N))` is 0 for small N (when log log N < 0, since `Real.sqrt` returns 0 for negative inputs). This is harmless since the bound only needs to hold eventually.
