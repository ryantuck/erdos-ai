## 6. Correctness

### Critical Bug: Vacuous Hypothesis at X = ‚ąÖ

The formalization has a **fatal correctness error** that makes the theorem trivially false (and therefore unprovable).

The degeneracy condition is:
```lean
‚ąÄ X : Finset (Fin n),
  ((F.filter (fun e => e ‚äÜ X)).card : ‚ĄĚ) < c t * (X.card : ‚ĄĚ)
```

When `X = ‚ąÖ`:
- No edge of size ‚Č• t (for t ‚Č• 1) is a subset of ‚ąÖ, so `(F.filter (fun e => e ‚äÜ ‚ąÖ)).card = 0`.
- `X.card = 0`, so `c t * 0 = 0`.
- The condition becomes `(0 : ‚ĄĚ) < 0`, which is **false**.

**Consequence:** The hypothesis `‚ąÄ X, ... < c t * |X|` is *never* satisfiable (it fails at X = ‚ąÖ). This makes the implication `hypothesis ‚Üí HasPropertyB F` vacuously true for all F. Therefore:

```
‚ąÉ c, Tendsto c atTop atTop ‚ąß (‚ąÄ t n F, [vacuously true])
```

is trivially satisfiable (pick any c with c ‚Üí ‚ąě). So the RHS of the biconditional is `True`, and the theorem becomes `False ‚ÜĒ True`, which is `False` ‚ÄĒ **unprovable**.

### Suggested Fix

Replace strict inequality `<` with `‚Č§`:

```lean
‚ąÄ X : Finset (Fin n),
  ((F.filter (fun e => e ‚äÜ X)).card : ‚ĄĚ) ‚Č§ c t * (X.card : ‚ĄĚ)
```

With `‚Č§`, when X = ‚ąÖ we get `0 ‚Č§ 0`, which is true, so the condition is no longer vacuously false. This also faithfully captures the intended degeneracy-type condition: every induced sub-hypergraph on X has at most $c_t |X|$ edges. The strict vs. non-strict distinction is immaterial for the truth of the conjecture (replacing $< c_t$ with $\leq c_t$ doesn't change whether such a $c_t \to \infty$ exists), and `‚Č§` avoids the degenerate edge case.

Alternatively, one could restrict X to nonempty sets:

```lean
‚ąÄ X : Finset (Fin n), X.Nonempty ‚Üí
  ((F.filter (fun e => e ‚äÜ X)).card : ‚ĄĚ) < c t * (X.card : ‚ĄĚ)
```

### Secondary Issue: Ground Set Generality

The formalization uses `Fin n` as the ground set, which means the theorem quantifies over all finite ground set sizes `n`. This is mathematically equivalent to the original (which says "every finite hypergraph") but is a slightly non-standard encoding. The approach in `ErdosProblems/901.lean` (using `‚Ąē` as vertices) avoids the dependently-typed `Fin n` and may be simpler to work with. This is a style preference, not a correctness issue.
