# Review: Erdős Problem 687

**File:** `FormalConjectures/ErdosProblems/687.lean`

## 1. Code Reuse

Several related formalizations exist in the codebase:

- **Problem 688** (`ErdosProblems/688.lean`) defines `PrimeCoveringInRange`, which is structurally almost identical to `PrimeCovering` but restricts primes to a range `(n^ε, n]` instead of all primes `≤ x`. The two predicates are similar enough that a shared, parameterized definition could serve both (e.g., a covering predicate taking a prime filter/predicate), but the specialization is simple enough that the duplication is arguably acceptable.

- **Problem 970** (`ErdosProblems/970.lean`) formalizes Jacobsthal's conjecture (`h(k) ≪ k²`), which 970.lean itself notes is "a more general form of the function considered in problem 687." The Jacobsthal function is defined via coprimality rather than residue class selection, so the formalization approach is genuinely different despite the deep mathematical connection. No direct code sharing is needed.

- **Problem 860** (`ErdosProblems/860.lean`) defines `HasPrimeCovering` for a different notion (injective assignment of multiples). Not reusable here.

- **`FormalConjecturesForMathlib/NumberTheory/CoveringSystem.lean`** defines `CoveringSystem` and `StrictCoveringSystem` for general commutative semirings. These are too general for direct use here — the 687 formalization needs specifically prime-indexed congruence classes with a bound on the primes, not an arbitrary finite index type with ideal-based moduli. Using `CoveringSystem` would add complexity without benefit.

**Verdict:** No significant code reuse opportunities. The duplication with 688 is minor and justified by the different prime-selection criteria.

## 2. Citations

The website ([erdosproblems.com/687](https://www.erdosproblems.com/687)) provides substantially more context than the current docstring:

- **Iwaniec's upper bound:** Y(x) ≪ x² (this is the starting point motivating the conjecture).
- **Ford, Green, Konyagin, Maynard, and Tao lower bound:** Y(x) ≫ x · (log x)(log log log x)/(log log x).
- **Maier–Pomerance conjecture:** Y(x) ≪ x(log x)^{2+o(1)}.
- **Prize:** $1000 (Erdős noted this was "$1000 plus half his life savings").
- **Related problems:** #4 (prime gaps), #688, #689, #970.
- **OEIS sequences:** A048670, A058989.

The current docstring includes none of these references. At minimum, the Iwaniec bound and the FGKMT lower bound should be cited, as they frame the gap the conjecture addresses. Cross-references to problems 688, 689, and 970 would also be valuable.

**Recommendation:** Add citations in the module docstring:
```
[Iw78] Iwaniec, H., *On the problem of Jacobsthal*. (upper bound Y(x) ≪ x²)
[FGKMT18] Ford, Green, Konyagin, Maynard, Tao, *Long gaps between primes*. (lower bound)

See also problems [#688], [#689], [#970].
```

## 3. Variants

The formalization captures:
- **Weak form** (`erdos_687`): Y(x) = o(x²) ✓
- **Strong form** (`erdos_687.variants.strong`): Y(x) ≤ x^{1+ε} for all ε > 0 ✓

**Missing from the website:**
- **Weaker variant:** The website mentions a variant where the covering is only required to hit all but o(y / log y) integers in [1, y], rather than all of them. This is not formalized.
- **Known bounds as solved variants:** Iwaniec's Y(x) ≪ x² and the FGKMT lower bound could be formalized as `category research solved` theorems, similar to how problem 688 formalizes its known lower bound.

## 5. Formalizability

**Assessment: Highly formalizable, low ambiguity.**

The problem statement defines Y(x) precisely as a maximum over covering functions and asks for asymptotic bounds. Both the weak form (o(x²)) and strong form (x^{1+o(1)}) have standard, unambiguous mathematical translations:

- Y(x) = o(x²) means: for all ε > 0, Y(x) < εx² for sufficiently large x.
- Y(x) ≤ x^{1+o(1)} means: for all ε > 0, Y(x) ≤ x^{1+ε} for sufficiently large x.

Both are captured correctly. The only minor ambiguity in the original problem is the phrasing "Y(x) ≪ x^{1+o(1)}" — the Vinogradov symbol ≪ technically allows a multiplicative constant, but since any constant C can be absorbed into the exponent (C · x^{1+ε} ≤ x^{1+2ε} for large x), the formalization's choice to omit the constant and use `≤` is mathematically equivalent and simpler.

The `answer(sorry)` wrapper correctly reflects the open status of the problem (the answer is not yet known).

## 6. Correctness

**Assessment: Correct and mathematically sound.**

**`PrimeCovering` definition (line 37–38):**
- `∀ n : ℕ, 1 ≤ n → n ≤ y → ∃ p : ℕ, p.Prime ∧ p ≤ x ∧ n % p = a p % p`
- This correctly states that every integer n ∈ {1, …, y} is congruent to a(p) mod p for some prime p ≤ x.
- The use of `n % p = a p % p` in `ℕ` correctly captures the congruence n ≡ a(p) (mod p), since both sides are reduced to the canonical representative in {0, …, p−1}.
- The function `a : ℕ → ℕ` is defined on all naturals, but only its values at primes ≤ x matter. This is a standard and harmless over-specification.

**Weak form (`erdos_687`, lines 46–52):**
- Correctly encodes Y(x) = o(x²): for every ε > 0, eventually Y(x) < εx².
- Uses strict inequality `<`, which is standard for little-o.

**Strong form (`erdos_687.variants.strong`, lines 59–65):**
- Correctly encodes: for every ε > 0, eventually Y(x) ≤ x^{1+ε}.
- Uses `≤` (non-strict), which is appropriate here.

**No mathematical flaws detected.** The formalization accurately captures the problem as stated on the website. The quantifier structure (∀ε > 0, ∃X, ∀x ≥ X, ∀ covering, bound on y) correctly expresses both asymptotic claims.
