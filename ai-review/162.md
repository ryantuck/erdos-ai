# Review: ErdŇĎs Problem 162

## 1. Code Reuse

**Problem 161 (`FormalConjectures/ErdosProblems/161.lean`) is nearly identical in structure.** Problem 162 is the special case t = 2 of Problem 161's t-uniform hypergraph setting. Specifically:

- `Erdos162.IsEdgeBalanced n m őĪ c` is essentially `Erdos161.IsBalancedColouring n 2 m őĪ c`, except for inequality strictness (`<` in 162 vs `‚Č§` in 161).
- `Erdos162.F n őĪ` mirrors `Erdos161.fBalanced 2 n őĪ` with the same `sInf` pattern.
- Both use `c : Finset (Fin n) ‚Üí Bool` for colorings and `powersetCard` for subset enumeration.

**Recommendation:** Problem 162's definitions could be expressed in terms of Problem 161's, or a shared parameterized definition could be factored out. At minimum, a comment noting the relationship would aid readability.

**Other reuse opportunities:**
- Problem 76 (`Erdos76.EdgeTwoColoring`) defines `Sym2 (Fin n) ‚Üí Bool`, a more precise type for edge 2-colorings. Problem 162 could use this instead of the over-specified `Finset (Fin n) ‚Üí Bool`.
- Problem 165 uses the same `(c - őĶ) * f(n) ‚Č§ g(n) ‚Č§ (c + őĶ) * f(n)` pattern for encoding `~` asymptotics and could share a common asymptotic definition.

## 4. Readability

**Generally good**, with a few suggestions:

- **Coloring domain:** The type `c : Finset (Fin n) ‚Üí Bool` assigns colors to *all* finsets of `Fin n`, not just 2-element subsets (edges). While this doesn't affect correctness (only 2-element subsets are queried via `powersetCard 2`), it is over-specified. Using `Sym2 (Fin n) ‚Üí Bool` (as in Problem 76) or restricting the domain would make the edge-coloring intent clearer.
- **Name consistency:** Problem 161 uses `IsBalancedColouring` / `fBalanced`; Problem 162 uses `IsEdgeBalanced` / `F`. The naming conventions differ despite the problems being closely related.
- **The `F` function name** is terse but matches the mathematical notation, which is acceptable.

## 5. Formalizability

**The problem is precise enough to be obviously formalizable.** The conjecture has a clear quantitative statement: F(n, őĪ) ~ c_őĪ log n. The definitions of F(n, őĪ) and "őĪ-balanced" are unambiguous once the direction of the quantifier (‚ąÉ vs ‚ąÄ over colorings) is fixed.

**Minor ambiguity in the source:** The original problem says "for a 2-colouring of the edges of K_n, let F(n, őĪ) be the smallest m..." The phrase "for a 2-colouring" could be read as "for every" (‚ąÄ) or "for some" (‚ąÉ). The formalization uses ‚ąÉ (there exists a coloring), which is the correct interpretation: F(n, őĪ) measures the best achievable threshold over all colorings. The ‚ąÄ interpretation would make F(n, őĪ) infinite (a monochromatic coloring is never balanced for őĪ > 0).

**Ambiguity level: Low.** The ‚ąÉ interpretation is the only mathematically meaningful one.

## 6. Correctness

### 6a. Range of őĪ ‚ÄĒ **BUG**

The formalization uses `0 ‚Č§ őĪ ‚Üí őĪ ‚Č§ 1 / 2 ‚Üí` (non-strict on both sides). This is problematic:

- **őĪ = 1/2:** The balanced condition requires each color class to have *strictly more than* (1/2) ¬∑ C(|X|, 2) edges. Since the two color classes partition all C(|X|, 2) edges, both cannot simultaneously exceed half. The condition is unsatisfiable for any coloring, making `{m | ‚ąÉ c, IsEdgeBalanced n m (1/2) c} = ‚ąÖ`, so `F n (1/2) = sInf ‚ąÖ = 0` (in Lean's ‚Ąē lattice). The conjecture then asks for `0 ~ c ¬∑ log n` with c > 0, which is false. **The conjecture is false at őĪ = 1/2.**

- **őĪ = 0:** The condition becomes `0 < |color class|`, i.e., each color class is non-empty. This is meaningful but may have different asymptotic behavior than the őĪ > 0 case (Problem 161 explicitly discusses how the őĪ = 0 case has iterated-logarithm growth for t ‚Č• 3).

**Compare with Problem 161**, which uses `0 < őĪ ‚Üí őĪ < 1 / 2 ‚Üí` (strict on both sides). Problem 162 should match: the range should be **0 < őĪ** and **őĪ < 1/2** (strict inequalities).

### 6b. Inequality direction (strict vs non-strict)

The formalization uses strict inequality `őĪ * C(|X|, 2) < |color class|` ("more than őĪ fraction"), while Problem 161 uses non-strict `őĪ * C(|X|, t) ‚Č§ |color class|` ("at least őĪ fraction"). The docstring says "more than an őĪ fraction," so strict is consistent with the stated intent. However, for the asymptotic conjecture, the difference between strict and non-strict is immaterial (it changes F by at most a constant). **Not a bug, but worth noting the inconsistency with Problem 161.**

### 6c. Asymptotic encoding

The formalization encodes `F(n, őĪ) ~ c_őĪ log n` as:

```
‚ąÉ c, 0 < c ‚ąß ‚ąÄ őĶ > 0, ‚ąÉ N‚āÄ, ‚ąÄ n ‚Č• N‚āÄ,
  (c - őĶ) * log n ‚Č§ F(n, őĪ) ‚Č§ (c + őĶ) * log n
```

This is equivalent to `F(n, őĪ) / log n ‚Üí c` as `n ‚Üí ‚ąě`, which is the standard definition of `~`. The `(c - őĶ)` vs `c(1 - őĶ)` formulation is equivalent since őĶ ranges over all positive reals. **Correct.**

### 6d. Definition of F via sInf

The use of `sInf` over `‚Ąē` is appropriate. If the set is empty (no coloring achieves balance at any threshold), `sInf ‚ąÖ = 0` in Lean. For őĪ ‚ąą (0, 1/2), any 2-coloring is balanced at threshold n (vacuously, since no subset has more than n elements), so the set is non-empty and `sInf` is well-defined. **Correct for the valid range of őĪ.**
