## 1. Code Reuse

The `erdosDelta` definition uses nested products with a conditional:
```lean
∏ i : Fin n, ∏ j : Fin n, if i ≠ j then ‖z i - z j‖ else 1
```

Two alternative patterns exist in the codebase for computing pairwise distance products:

- **Erdős 1040** (`FormalConjectures/ErdosProblems/1040.lean:48–50`): Uses `Finset.filter (fun p => p.1 < p.2)` on `univ : Finset (Fin n × Fin n)` to compute `∏_{i<j}`.
- **Erdős 991** (`FormalConjectures/ErdosProblems/991.lean:49–50`): Uses `Finset.offDiag` to compute `∏_{i≠j}`.

The `offDiag` pattern from Problem 991 computes the same quantity as `erdosDelta` (product over ordered pairs with i ≠ j) and is arguably more idiomatic Mathlib style. However, since 991 works with `Finset` rather than `Fin n → ℂ`, the definitions are not directly interchangeable without adaptation. No shared utility exists in `FormalConjecturesForMathlib/` for this pattern; each problem defines its own helper. The `regularNGon` definition is unique to this file and not reused elsewhere.

**Recommendation:** The current self-contained approach is acceptable. If code sharing were desired, a common `pairwiseDistProd` utility for `Fin n → ℂ` configurations could be factored out, but this is low priority.

## 6. Correctness

**Assessment: Correct and complete.**

**Definition `erdosDelta`:** Computes ∏_{i≠j} ‖z_i − z_j‖ by iterating over all ordered pairs (i, j) with i ≠ j, substituting 1 for i = j terms. This equals (∏_{i<j} |z_i − z_j|)². Since the same definition is used on both sides of the inequality, the squaring is harmless and the comparison is mathematically equivalent to the original problem. ✅

**Definition `regularNGon`:** Produces e^{2πik/n} for k = 0, …, n−1, which are the vertices of a regular n-gon on the unit circle (radius 1, diameter 2). All pairwise distances are ≤ 2 since all points lie on the unit circle. ✅

**Main theorem (`erdos_1045`):** States `answer(False) ↔ (∀ n ≥ 1, ∀ z with constraint, erdosDelta z ≤ erdosDelta (regularNGon n))`. Since `answer(False) = False`, this asserts the universal statement is false, i.e., ∃ n ≥ 1 and z satisfying the constraint with erdosDelta z > erdosDelta (regularNGon n). This correctly captures the disproved conjecture (counterexamples exist for n = 4, 6, and all even n ≥ 4). Category `research solved` is correct. ✅

**Odd variant (`erdos_1045.variants.odd`):** States that for all odd n ≥ 1, the regular n-gon is optimal. Uses `answer(sorry)` reflecting that the truth value is unknown. Category `research open` is correct. ✅

**Edge cases:**
- n = 0: `Fin 0 → ℂ` is the unique empty function. `erdosDelta 0 z` is an empty product = 1. The main theorem requires n ≥ 1, so this is excluded. ✅
- n = 1: Both sides give empty product = 1. Trivially satisfied. ✅
- n = 2: `regularNGon 2` gives {1, −1} with distance 2. `erdosDelta = 4`. Any z with |z₁ − z₂| ≤ 2 gives erdosDelta = |z₁ − z₂|² ≤ 4. Optimal. ✅
